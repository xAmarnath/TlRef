Errors

Every RPC error gogram knows about, grouped by HTTP-style status and topic. Each entry shows the wire code, what the server says, and a short explanation of when and why it usually fires.

606 RPC errors
18 Parameterized
11 MTProto bad-msg codes

Source of truth: errors.go in gogram. In Go these surface as *gogram.ErrResponseCode — match on .Message for the exact code, read .Description for the formatted message, and inspect .AdditionalInfo for the parsed parameter when the code ends in _X.

The HTTP code is the one Telegram returns alongside the error. 420 means the account is being rate-limited and you must wait. 303 means the server is telling you to talk to a different data center; gogram handles that transparently. 401 means the session is gone and the user has to re-authenticate.

303 See Other 5 errors

The auth key/account lives on a different DC. The gogram client transparently reconnects and replays the request.

MTProto Bad-message notifications 11 codes

Service-message error codes returned by the MTProto transport itself, before any RPC dispatch happens. Usually the client's clock is wrong, the session salt expired, or a message id collided. Gogram surfaces these as *BadMsgError.

code 16

msg_id too low (most likely, client time is wrong; it would be worthwhile to synchronize it using msg_id notifications and re-send the original message with the “correct” msg_id or wrap it in a container with a new msg_id if the original message had waited too long on the client to be transmitted)

code 17

msg_id too high (similar to the previous case, the client time has to be synchronized, and the message re-sent with the correct msg_id

code 18

incorrect two lower order msg_id bits (the server expects client message msg_id to be divisible by 4)

code 19

container msg_id is the same as msg_id of a previously received message (this must never happen)

code 20

message too old, and it cannot be verified whether the server has received a message with this msg_id or not

code 32

msg_seqno too low (the server has already received a message with a lower msg_id but with either a higher or an equal and odd seqno)

code 33

msg_seqno too high (similarly, there is a message with a higher msg_id but with either a lower or an equal and odd seqno)

code 34

an even msg_seqno expected (irrelevant message), but odd received

code 35

odd msg_seqno expected (relevant message), but even received

code 48

incorrect server salt (in this case, the bad_server_salt response is received with the correct salt, and the message is to be re-sent with it)

code 64
