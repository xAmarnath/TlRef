E2E Schema

Telegram's end-to-end encrypted (secret-chat) TL schema. These constructors travel inside the encrypted payload of a regular MTProto message, separate from the main API — every secret chat agrees on its own layer at handshake time and uses these types for everything sent after.

90 Constructors
14 Abstract types
0 Methods
Layers 8–216

How the E2E schema fits in

Secret chats use a Diffie–Hellman handshake to derive a shared auth_key. Every secret-chat message you'd otherwise pass to messages.sendEncrypted is first serialised as a DecryptedMessageLayer wrapping a DecryptedMessage from this schema, then AES-256-IGE encrypted with that key.

Each abstract type below (e.g. DecryptedMessage, DecryptedMessageMedia) is a tagged union — the wire id of the chosen constructor selects which variant to decode. Layers only ever add variants, so old peers can still decode messages, but new fields are dropped.

Source: core.telegram.org/schema/end-to-end-json. Constructors inside each group are sorted by the layer they were introduced in.
